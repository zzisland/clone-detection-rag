NiCad克隆检测工具使用指南

NiCad是一个功能强大的代码克隆检测工具，支持多种编程语言。

安装要求：
- Java 8或更高版本
- 足够的内存（建议2GB以上）

基本使用方法：

1. 配置文件设置
创建nicad.properties文件：
src_dir = 源代码目录
language = java/c/python/etc
clone_type = 1/2/3/4
granularity = function/class/file
threshold = 相似度阈值（0-1）

2. 运行命令
java -jar nicad.jar [选项] 配置文件

常用选项：
- -v：详细输出
- -r：递归搜索
- -o：输出目录

3. 输出格式
- XML格式：包含克隆对的详细信息
- HTML报告：可视化展示克隆结果
- CSV格式：便于数据分析

克隆类型设置：
- Type-1：精确克隆检测
- Type-2：语法克隆检测  
- Type-3：语义克隆检测
- Type-4：功能克隆检测

粒度设置：
- function：函数级别检测
- class：类级别检测
- file：文件级别检测

阈值建议：
- Type-1：0.9-1.0
- Type-2：0.8-0.9
- Type-3：0.6-0.8
- Type-4：0.4-0.6

性能优化：
1. 使用合适的粒度级别
2. 设置合适的阈值
3. 限制搜索深度
4. 排除测试文件

常见问题：
1. 内存不足：增加JVM堆内存
2. 检测速度慢：使用更粗粒度或提高阈值
3. 误报率高：调整阈值或使用更严格的克隆类型

最佳实践：
1. 先使用Type-1快速扫描
2. 逐步放宽到Type-2、Type-3
3. 结合人工审查确认结果
4. 定期更新配置以适应代码变化
