JPlag克隆检测工具使用指南

JPlag是一个基于结构指纹的代码相似度检测工具，专为教育环境设计。

特点：
- 教育导向：专为学术环境优化
- 多语言：支持多种编程语言
- 结构化：基于程序结构分析
- 可配置：灵活的参数设置

安装要求：
- Java 8或更高版本
- 至少1GB可用内存
- 网络连接（可选，用于更新）

基本语法：
jplag [选项] 源代码目录

主要选项：
- -l：指定编程语言
- -t：设置相似度阈值
- -r：指定结果目录
- -s：最小匹配大小
- -m：匹配算法
- -v：详细输出

语言支持：
- java：Java语言
- c：C语言
- cpp：C++语言
- python：Python语言
- csharp：C#语言
- php：PHP语言
- ruby：Ruby语言
- text：纯文本
- scheme：Scheme语言

使用示例：
1. 检测Java作业：
jplag -l java -t 0.3 -r results submissions/

2. 高精度检测：
jplag -l java -t 0.7 -s 10 -r results submissions/

3. Python代码检测：
jplag -l python -t 0.4 -r results python_submissions/

4. 详细输出模式：
jplag -l java -t 0.3 -v -r results submissions/

匹配算法：
- 默认算法：基于结构指纹
- 快速算法：基于token序列
- 精确算法：基于语法树
- 自定义算法：用户定义

阈值设置：
- 严格检测：0.7-1.0
- 标准检测：0.3-0.7
- 宽松检测：0.1-0.3
- 自定义：根据需要调整

结果格式：
HTML报告：
- 总体统计信息
- 相似度分布图
- 文件配对详情
- 相似代码片段
- 可视化展示

XML报告：
- 结构化数据
- 机器可读
- 易于解析
- 详细统计

CSV报告：
- 表格格式
- 数据分析友好
- 易于导入
- 简洁明了

配置文件：
创建jplag.properties：
language=java
threshold=0.3
result_dir=results
min_match=10
verbose=true
submissions=submissions/

性能优化：
1. 合理设置阈值
2. 选择合适的算法
3. 排除不相关文件
4. 增加内存设置

集成方式：
1. 命令行使用：
直接运行jplag命令

2. Web界面：
使用内置Web服务器

3. API集成：
通过编程接口调用

4. IDE插件：
集成到开发环境

1. 内存不足：增加JVM内存
2. 检测速度慢：调整算法或阈值
3. 误报率高：提高阈值或调整参数
4. 结果异常：检查输入数据质量
